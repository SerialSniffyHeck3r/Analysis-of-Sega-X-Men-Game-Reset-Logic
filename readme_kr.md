# Sega Mega Drive용 X-MEN의 `RESET THE COMPUTER!` 기믹 분석 (KR)

이 문서는 Sega Mega Drive(Genesis)용 게임 **X-MEN (1993)** 에 등장하는,  
악명 높은 퍼즐 구간 **“RESET THE COMPUTER!”** 가 **하드웨어·소프트웨어 관점에서 어떻게 구현되어 있는지**를 정리한 기술 노트입니다.

- 대상 게임: X-MEN (Sega, 1993, Mega Drive/Genesis)
- 주제: “RESET THE COMPUTER!” 퍼즐을 위해 실제 콘솔 **RESET 버튼**을 사용하게 만든 메커니즘
- 관점:
  - Mega Drive 리셋 신호 구조
  - 메인 RAM 특성 (리셋 시 데이터 유지)
  - X-MEN 롬 디스어셈블 결과 (매직 넘버 검사 루틴)
  - 퍼즐이 실제로 어떻게 동작하는지에 대한 전체 플로우

이 문서는 실제 공개된 하드웨어 분석 및 디스어셈블 자료를 참고한 개인 연구로써,

실제 선행 연구를 글로써 요약한 버전입니다.

---

## 문제의 퍼즐: “RESET THE COMPUTER!”

게임 후반, Mojo 스테이지를 클리어한 뒤 마지막 스테이지로 넘어가기 직전에  
플레이어는 다음과 같은 상황을 마주할 수 있다.

- 화면에는 게임 내 “컴퓨터/단말기” 오브젝트가 보이고  
- 캐릭터들은 계속해서 **“Reset the computer!”** 라고 말한다.
- 그러나 게임 내 메뉴, 옵션, 버튼 어디에서도 “컴퓨터를 리셋하는 조작”은 제공되지 않는다.
- 화면은 그저 컴퓨터를 리셋하라는 것에서 멈춰 잇다.

당시 설명서에도 이 퍼즐을 푸는 방법은 나와 있지 않았기 때문에,  
많은 플레이어들이 이 지점에서 진행이 완전히 막혔다고 보고한 바는 유명하다.

일부 잡지 공략이나 입소문을 통해서만 정답이 알려졌고,  
그 정답은 정말로 직관과 정반대였다.

어쩌면, 그 누구도 보통의 상식으로 예측하기 어려운 그 행위란 바로,

> **정답: 실제 Mega Drive 본체의 RESET 버튼을 눌러야 한다.**

여기서도 조건이 하나 더 붙는다.

- RESET 버튼을 **짧게** 눌러야 한다.
- 버튼을 길게 눌러 “완전한 리셋”이 일어나거나,  
  전원을 껐다 켜 버리면 게임은 처음부터 다시 시작된다.

이 퍼즐은 오늘날까지도 “하드웨어를 이용해 진행을 거는,  
매우 비직관적인 게임 디자인”의 대표적인 사례로 자주 언급되는 바가 있다.

개인적으로, 이 게임 디자인은 여러 면에서 인상깊다.
해당 스테이지는 게임 전체를 통틀어 어려운 레벨이며, 또한 게임 진행의 후반부이기도 하다.
리셋 버튼을 누르는 것은 저장 없이 이것을 통째로 리셋해버리겠다는 것을 의미한다고 받아들일 수 있으며,

어떻게 보면 바로 이 기믹은 '플레이어의 깡을 시험하는' 의도된 레벨 디자인일지도 모른다.

아무튼... 이 기믹은 분명히 인상 깊다. 그 해답이 주는 신선하고 맛있는 충격 뿐만 아니라,
기술적인 면에서도 마찬가지다. 그니까 리셋을 했는데 어떻게 게임이 진행되지? 하는 것 말이다.

## 2. Mega Drive의 리셋 신호 구조

### 2.1 리셋 버튼의 기본 동작

Mega Drive의 RESET 버튼은 메인보드 상의 리셋 신호 라인에 연결된  
**디지털 입력 스위치**이다.

- 평소(버튼이 눌리지 않았을 때)
  - 해당 라인은 풀다운 저항에 의해 **논리 0(LOW)** 로 유지됩니다.
- 버튼을 누르는 순간
  - RESET 라인은 **논리 1(HIGH)** 로 상승하며,  
    “리셋 요청이 발생했다”는 의미가 된다.

일반적인 기계식 스위치와 마찬가지로, 짧은 시간 동안 여러 번 튕기는  
**스위치 바운스(bounce)** 현상이 존재하지만,  
리셋 라인의 특성상 **몇 번 더 리셋이 들어간다고 해서 큰 문제가 발생하지 않도록**  
설계되어 있기 때문에 별도의 디바운스 회로 없이 사용되는 것으로 알려져 있으며,
이는 일종의 원가 절감 사례로 생각되나 이 설계의 정확한 이유는 알려져 있지 않다.

### 2.2 여러 종류의 리셋 신호

Mega Drive에는 내부적으로 여러 종류의 리셋 신호가 존재하는데, 

- `WRES`  
  - 콘솔 전면의 **RESET 버튼에서 직접 나오는 신호**
  - Active High
  - 평소에는 LOW, 버튼을 누르면 HIGH
  - 별도의 디바운싱 없이 사용
- `MRES`  
  - 파워 온 시퀀스를 담당하는 **메인 리셋 (power-on reset)**  
  - RC 회로(저항 + 커패시터)를 통해 전원 인가 시 일정 시간 동안 리셋을 유지하는 역할
- `SRES`  
  - `MRES` 를 기반으로 생성되는 보조 리셋 신호
  - 일부 커스텀 칩(VDP, I/O 컨트롤러 등)에 먼저 전달되어,
    68000 CPU가 동작하기 전에 주변 칩들을 초기 상태로 만든다.
- `VRES`  
  - 68000 CPU의 `/RESET` 입력으로 들어가는 리셋 신호
  - `WRES`(버튼) 또는 `SRES`(전원 인가)에서 유도되어,
    **정해진 길이(예: 128 VCLK 사이클 동안 LOW)** 로 펄스를 생성한다.
- 기타 `ZRES`, `FRES` 등  
  - Z80 및 추가 장치(Mega CD 등)를 리셋하는 데 사용되는 신호들

중요한 점은...

> RESET 버튼을 누르면, `WRES` 가 HIGH가 되고  
> 이 입력을 기반으로 `VRES` 가 **짧은 펄스**로 생성되어  
> 68000 및 관련 칩들이 **안정적인 타이밍으로 동시에 리셋된다는 것** 이다.

이 과정에서 **전원은 계속 유지된다는 점이 중요하다.**  
전원은 계속 보드 전체에 공급되고 있고,  
리셋 신호는 어디까지나 **“칩들의 상태를 초기화하라”**는 논리 신호일 뿐이다.

## 3. 메인 RAM과 리셋: 왜 데이터가 남을까?

### 3.1 Mega Drive 메인 RAM의 종류

Mega Drive의 메인 CPU(68000)는 **64 KiB의 메인 RAM**을 사용하는데...
이 RAM은 단순한 정적 SRAM이 아니라,  
내부에 리프레시 로직이 포함된 **PSRAM(의사(static) RAM)** 이 사용된 것으로 알려져 있다.

PSRAM은 외부 인터페이스는 SRAM처럼 보이지만,  
실제 내부 구현은 DRAM과 유사해 별도의 리프레시 회로가 포함되어 있다는 점이 차이점이다.

아무튼... 으레 대부분의 램이 그렇듯, 전원이 유지되는 한, 이 리프레시 회로 덕분에 데이터가 계속 유지된다.

### 3.2 리셋과 전원 차단의 차이

여기서 핵심 포인트는 다음과 같다.

- **리셋 버튼**
  - 68000과 주변 칩들의 상태를 초기화
  - 프로그램 카운터를 리셋 벡터로 되돌리고,  
    메모리 맵을 초기 상태로 되돌리는 역할
  - 전기적인 수준에서 전원을 끊지 않으므로, **RAM의 내용을 지우지 않는다**
- **전원 스위치 OFF**
  - 보드 전체에 공급되는 전원을 차단
  - PSRAM 내부 커패시터 및 회로가 방전되면서  
    **RAM 내용이 진짜로 휘발**

따라서 게임 카트리지 입장에서 보면:

> RESET 버튼을 눌렀을 때는,  
> **“처음부터 다시 시작하되 RAM은 그대로인 상태”** 에서 실행이 재개되고,
> 그 게임 카트리지의 엔트리 포인트부터 시작되는 초기화 시퀀스가 램을 초기화한다.

즉, 게임 코드가 부트 시점에 **RAM의 특정 위치를 검사**해  
“이전 실행에서 남겨둔 서명(signature)이 있는지”를 확인하는 것이 기술적으로 충분히 가능하고,
해당 게임은 은 이 특성을 적극 활용한다.

## 4. X-MEN 롬 디스어셈블: 매직 넘버 검사

### 4.1 부트 진입점 근처의 코드

커뮤니티에서 공개된 디스어셈블 결과에 따르면,  
X-MEN 롬에는 리셋 직후 실행되는 루틴(편의상 `Start2`) 초반에  
다음과 같은 코드가 등장한다.

```asm
Start2:
    ; ... (여러 초기화 루틴 호출)

    cmpi.l  #$57425554, $00FF0800.l  ; RAM 0xFF0800에 저장된 값이 0x57425554인지 비교
    bne.b   @notmagic                ; 다르면 일반 부팅으로 분기

    jmp     $027FE0                  ; 같으면 특수 루틴(엔드game 관련)으로 점프

@notmagic:
    ; 이후에는 일반적인 초기화 및 게임 시작 루틴 진행
    ; ...
```

여기서 사용된 상수 `0x57425554`는  
ASCII 문자열로 해석하면 `"WBUT"`에 해당.

- 의미 있는 단어는 아니며,
- **게임 내부에서 특별한 상태를 나타내기 위한 “매직 넘버”** 로 쓰인 것으로 해석된다.
- 디스어셈블 결과에서는 이 비교가 **리셋 직후 매우 이른 시점**에 수행된다는 점이 중요하다.

### 4.2 주소 0xFF0800의 역할

위 코드에서 비교하는 주소는 **`0xFF0800`** 입니다.

- 이 주소는 Mega Drive의 메인 RAM 영역 안에 위치한다.
- 전원이 유지되는 동안 리셋만 반복해도,  
  이 위치에 저장된 데이터는 그대로 남아 있다.
- 즉, 게임은 퍼즐 구간에서 **이 위치에 `"WBUT"` 값을 기록**해 두고,  
  이후 리셋이 들어온 뒤 다시 부팅할 때  
  “이 값이 남아 있는지”를 검사할 수 있다.

## 5. “RESET THE COMPUTER!” 퍼즐의 동작 흐름

### 5.1 퍼즐 구간에서 일어나는 일 (게임 내부 관점)

퍼즐 구간에서, 게임은 내부적으로 대략 다음과 같은 작업을 수행한다.

1. 플레이어가 특정 조건(해당 스테이지를 사실상 클리어한 상태)에 도달하면,
2. 메인 RAM의 주소 `0xFF0800` 에  
   **매직 넘버 `0x57425554` (`"WBUT"`) 를 기록**한다
3. 이후에는 게임 진행을 고의적으로 **막아 둔 채**,  
   화면과 대사를 통해 플레이어에게 계속해서  
   **“Reset the computer!”** 라는 스크린 그래픽을 띄운다
4. 이 시점에서 게임은 **플레이어가 콘솔 RESET 버튼을 누를 것**을 전제로 한다.

이 상태에서 플레이어가 RESET 버튼을 누르면:

- 하드웨어 레벨에서:
  - `WRES` 가 HIGH가 되고
  - 이를 기반으로 `VRES` 가 LOW 펄스를 생성하여  
    68000 및 관련 칩들이 리셋
- 그러나:
  - **전원은 그대로 유지되므로**,  
    메인 RAM에 기록된 `"WBUT"` 값은 계속 유지된다.

### 5.2 리셋 이후 부트 시퀀스

리셋이 끝나면, CPU는 다시 카트리지의 리셋 벡터에서부터 실행을 시작하는데..  
이때 X-MEN의 부트 루틴은 앞서 본 것처럼 다음 검사를 수행한다.

```asm
cmpi.l  #$57425554, $00FF0800.l  ; 0xFF0800 == "WBUT" ?
bne.b   @notmagic                ; 아니면 일반 부팅
jmp     $027FE0                  ; 맞으면 특수 루틴(엔드game)으로 점프
```

- 만약 `0xFF0800` 에 `"WBUT"` 가 남아 있다면:
  - 게임은 이를 “퍼즐을 올바르게 해결한 상태”로 간주하고  
  - **마지막 스테이지(마그네토가 있는 엔딩 구간)** 으로 바로 점프한다.
- 값이 맞지 않으면(전원을 껐다 켜서 RAM이 날아갔거나, 다른 상황):
  - 평소처럼 일반 타이틀/초기화 루틴으로 진행하게 된다.

결국 전체 퍼즐은 다음과 같이 요약할 수 있다.

> 1. 퍼즐 구간 진입 → RAM에 `"WBUT"` 기록  
> 2. 콘솔 RESET 버튼을 눌러 CPU 리셋 (전원은 그대로)  
> 3. 부트 루틴이 `"WBUT"` 존재 여부 확인  
> 4. 존재하면 → 마지막 스테이지로 점프

## 6. 왜 “리셋을 길게 누르면 진행이 날아간다”는 말이 나왔을까?

Reddit 등의 게임 관련 커뮤니티를 확인하면, 당시 플레이어들의 증언 중에는 다음과 같은 이야기가 자주 등장한다.

- RESET 버튼을 **살짝만 튕기듯이** 누르면 퍼즐이 정상 동작하는데,
- **너무 길게** 누르고 있으면 게임이 완전히 초기화되어  
  처음부터 다시 시작된다는 것.

기술적인 관점에서 이 현상을 해석해 보면, 몇 가지 가능성이 충분히 있다고 볼 수 있다.

### 6.1 “버튼 길이”를 게임이 직접 측정하는 것은 아님

공개된 분석에 따르면, X-MEN은 리셋 시나리오를 구분하기 위해  
**리셋 버튼이 얼마나 길게 눌렸는지** 를 직접 측정하는 것은 기술적으로 불가능하다.

- RESET 버튼이 눌리는 순간, 68000은 어차피 리셋되고,
- 그 이후의 실행 흐름은 **항상 동일한 리셋 벡터 → 부트 루틴 → X-MEN 초기 코드**,
- 게임이 감지하는 것은 오직 **“지금은 리셋 이후 실행이고, RAM 특정 위치에 특정 값이 남아 있는가?”** .

따라서 “길게 눌렀을 때만 따로 인식한다”기보다,  
다음과 같은 간접적인 요인이 작용했을 가능성이 크다고 볼 수 있다.

### 6.2 전원 공급 및 커패시터 방전

일부 하드웨어 분석에서는,  
RESET 버튼이 눌려 있는 동안 보드 상의 전원 배선이나  
일부 회로에 미묘한 영향이 갈 수 있다는 가능성도 제기한다.

- 극단적인 경우, 버튼을 아주 오랫동안 누르고 있거나
- 전원 상태가 좋지 않은 콘솔에서는
  - 보드 곳곳의 전압이 일시적으로 떨어지면서
  - RAM이나 리셋 관련 회로가 **실질적인 “소프트 전원 차단”과 유사한 상태**를 겪을 수도 있다.
- 이 경우 RAM 내용이 불안정해지거나,  
  특정 타이밍에서 초기화 코드가 RAM을 덮어쓰는 사태가 발생하는 것이 가능하다.

즉, 플레이어 입장에서는

> “짧게 누르면 잘 되는데, 길게 누르니까 진행이 날아갔다”  

는 경험을 하게 되지만,  
이는 게임이 버튼 길이를 인식해서 다르게 동작한 결과라기보다는  

- **전원 상태 / 하드웨어 편차 / 스위치 바운스 / 리셋 타이밍** 등  
  다양한 요인이 겹친 결과일 가능성이 높다.

### 6.3 스위치 바운스 및 반복 리셋

또 다른 가능성은 **스위치 바운스 및 반복 리셋**입니다.

- RESET 버튼을 깊게, 오래 누르는 과정에서
  - 리셋 신호가 여러 번 On/Off 되거나
  - 의도치 않은 타이밍에 리셋이 다시 걸리면서
- 부트 루틴의 일부가 **예상치 못한 순서로 여러 번 실행**될 수 있다.

이 과정에서:

- `"WBUT"` 를 기록하는 시점과,
- RAM을 초기화하거나 덮어쓰는 다른 루틴의 실행 순서가 꼬이면

리셋 이후 `"WBUT"` 가 보존되지 못할 수 있다.

정리하면:

> “길게 누르면 진행이 날아간다”는 현상은  
> **하드웨어, 전원, 타이밍 요인이 복합적으로 작용한 결과**일 가능성이 크며,  
> 게임 코드가 의도적으로 “길게 눌렀다”는 것을 구분해서 처리하는 구조는 아니다.

## 7. Genesis Nomad에서는 클리어 불가능했던 이유

흥미로운 후일담으로,  
이 퍼즐은 **Genesis Nomad** (메가드라이브 호환 휴대용 콘솔)에서  
사실상 클리어가 불가능하다는 문제도 있었다.

- Nomad는 Mega Drive 카트리지를 그대로 사용할 수 있지만,
- 본체에 **RESET 버튼이 존재하지 않는다.**
- 따라서 “RESET THE COMPUTER!” 구간까지 진행해도  
  퍼즐을 풀 수 있는 방법이 물리적으로 없었다는 것이다.

결과적으로 Nomad 사용자들은

> - 퍼즐 구간까지는 갈 수 있지만  
> - 그 이후로는 절대 마지막 스테이지에 도달할 수 없는

상황에 처하게 되었고,  
이 점은 X-MEN의 리셋 퍼즐이 **하드웨어 의존성이 너무 강한 기믹**이라는 비판의 근거가 되기도 했지만,

어차피 해당 게임은 메가 드라이브 전용으로 출시한 게임이었으므로 개인적으로는 상관이 없으며,
개인적으로도 매우 인상 깊은 기믹이라고 생각한다.

## 8. 정리: 하드웨어 특성을 이용한 메타 퍼즐

여기까지 내용을 종합하면,  
X-MEN의 “RESET THE COMPUTER!” 구간은 다음과 같은 특징을 갖는다고 정리할 수 있다.

1. **리셋 버튼은 메모리를 지우지 않는다.**
   - Mega Drive의 RESET 버튼은 68000과 주변 칩들의 상태를 초기화하지만,
   - 메인 RAM(PSRAM)은 **전원이 유지되는 한** 내용을 그대로 유지한다.

2. **게임은 이 특성을 적극 활용한다.**
   - 퍼즐 구간에서 RAM 특정 주소(0xFF0800)에 `"WBUT"` 매직 넘버를 기록해 두고,
   - 리셋 이후 부트 루틴에서 해당 값을 검사해  
     **“퍼즐을 클리어한 상태인지”** 판별한다.

3. **RESET 버튼을 퍼즐의 일부로 끌어들인 메타 연출**
   - 게임 내에서는 “컴퓨터를 리셋하라”고 말하지만,
   - 실제로는 **콘솔 본체의 RESET 버튼**을 눌러야만 진행 가능하도록 설계되어 있다.
   - 이는 하드웨어와 소프트웨어의 경계를 일부러 흐리게 만든  
     메타적 퍼즐 기믹이라 볼 수 한다.

4. **동시에, 디자인적으로는 논란의 대상**
   - 당시 플레이어 입장에서는
     - RESET 버튼은 “누르면 진행이 날아가는 위험한 버튼” 이라는 인식이 강했고,
     - 설명서나 게임 내 튜토리얼에서도 정답이 제시되지 않았기 때문에,
   - 많은 사람들이 이 퍼즐을 “사실상 진행불가 버그처럼 느껴지는 구간”으로 기억되고 있다.
   - 또한 Genesis Nomad에서는 아예 RESET 버튼이 없기 때문에,  
     물리적으로 클리어 불가능한 게임이 되어버린다는 점은 확실한 문제였다.

그럼에도 불구하고, 이 기믹은

> **“하드웨어 리셋 구조 + RAM 유지 특성 + 매직 넘버 검사”**

라는 요소를 조합해,  
하나의 퍼즐/연출로 완성시킨 독특한 사례로 남아 있다.

## 9. 참고용 요약 (TL;DR)

- Mega Drive RESET 버튼은 **CPU와 칩들을 리셋할 뿐, RAM을 지우지 않는다.**
- X-MEN은 퍼즐 구간에서 RAM 주소 `0xFF0800` 에 `"WBUT"` 라는 값을 기록해 둔다.
- 플레이어가 RESET 버튼을 누르면:
  - 전원은 그대로, RAM도 그대로
  - 부트 루틴 재실행 → X-MEN 초기 코드 진입
- 이때 X-MEN은 `0xFF0800 == "WBUT"` 인지 검사해
  - 맞으면 마지막 스테이지로 점프
  - 아니면 평소처럼 처음부터 시작
- 이 구조 덕분에, **현실의 RESET 버튼을 퍼즐의 해답으로 사용하는 메타 기믹**이 구현된다.

이 문서는 위와 같은 동작을  
Mega Drive 하드웨어 구조와 X-MEN 롬 디스어셈블 결과를 바탕으로 정리한 것입니다.
